# Multi-Agent Coordination Template

这个模板给协调人使用。目标不是让两个 Agent 同时自由发挥，而是让他们围绕同一份契约协作。

## 推荐流程
1. 先把需求拆成前端范围和后端范围。
2. 指定每个 Agent 可修改的目录或文件。
3. 让后端 Agent 先产出接口契约草案。
4. 把契约草案发给前端 Agent，请其确认是否足够实现。
5. 来回同步，直到契约稳定。
6. 契约冻结后，再让双方分别实现。
7. 最后统一联调、测试和合并。

## 协调规则
- 两个 Agent 不同时改同一个文件。
- 公共契约只允许一个 Agent 修改，另一个只能提意见。
- 不允许任何一方“先猜着写”对方的部分。
- 发现契约冲突时，先回到契约层解决，再继续编码。

## 你发给前端 Agent 的附加说明
- 只实现前端。
- 严格按契约消费接口。
- 如果契约缺字段、缺错误码、缺示例，直接提出，不要自行脑补。

## 你发给后端 Agent 的附加说明
- 先定义契约，再写实现。
- 响应结构、错误码、分页/排序/筛选规则必须写清楚。
- 给前端稳定字段，不随意改名。

## 建议的协商轮次
### 第一轮
- 前端提出需要哪些数据和交互。
- 后端给出接口草案。

### 第二轮
- 前端检查草案是否足够落地。
- 后端补充缺失字段、错误码和边界行为。

### 第三轮
- 双方确认契约冻结。
- 分别实现并回传验证结果。

## 最终检查清单
- 契约是否唯一且最新
- 两个 Agent 是否修改了互斥文件
- 前后端是否使用同一组字段名
- 错误码和空态是否对齐
- 联调步骤是否清晰
