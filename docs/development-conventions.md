# 通用开发约定（模板版）

> 本文档收录与具体技术栈解耦的开发约定 —— TDD 判定 / 调研降级 / 文件大小判据 / 分层验证 / 依赖策略。
>
> 下游 fork 模板后可直接采纳本文，或挑相关节 paste 到自家 `docs/`。模板自身**不引用**此文档作强制规则；模板的核心治理由 AGENTS.md + [adapters/README.md](../adapters/README.md) + 模板自家的 `./tasks.sh validate` 兜底。
>
> 占位符 `{{...}}` 由下游按自家栈替换（如 `{{validate命令}}` → `pnpm validate` / `make test`）。

## TDD 判定

判据：**把实现改错时，对应测试会失败**。豁免：仅注释 / doc / 类型签名内部整理 / 无行为重命名 / 常量 / re-export / ≤3 行纯路径拼接。禁止为依赖库固有能力（如内建 parse、数组 push）写测试。

## 新功能 / 重构调研

新功能或多文件重构前先调研同类成熟项目，避免闭门造车。**降级 ≠ 跳过**，仅以下条件可触发：用户限时 / 环境不可用 / 本地资料已覆盖。**不得自行判断「时间紧」**作为跳过理由；降级时记录触发条件和本地依据。

## 文件大小

拆分判据：**单一职责 + 可读性，不是行数**。Schema 集中定义、递归渲染器、生成产物等单一职责文件不强行拆分。

## 分层验证策略

- **不要每次跑全量**：内循环用**受影响的目标测试**；收口在 Phase 结束时跑一次 `{{validate命令}}`。
- **派发 subagent**：prompt 写明**目标测试 + 收口要求**；主流程**亲自跑收口 + 读退出码**，不只引用 subagent 报告。

## 依赖策略

- 优先复用现有依赖；解析 / 校验 / 日期 / CLI / 重试 / 日志 / schema / HTTP client / 文件匹配等通用能力**不要重复造轮子**。
- 新增依赖：提交说明写**选择原因 + 至少一个备选方案及未采用原因 + 运行时/维护成本/复杂度权衡**。
- **lockfile 必须提交**（`package-lock.json` / `pnpm-lock.yaml` / `yarn.lock` / `Cargo.lock` / `poetry.lock` / `uv.lock`），禁止 gitignore。
- 依赖升级**单独提交**，注明升级原因和影响范围；禁止在功能 PR 中混入无关升级。
