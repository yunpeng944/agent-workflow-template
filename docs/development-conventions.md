# 通用开发约定（模板版）

> 本文档收录与具体技术栈解耦的开发约定 —— TDD 判定 / 调研降级 / 文件大小判据 / 分层验证 / 依赖策略。
>
> 下游 fork 模板后可直接采纳本文，或挑相关节 paste 到自家 `docs/`。模板自身**不引用**此文档作强制规则；模板的核心治理由 AGENTS.md + [adapters/README.md](../adapters/README.md) + 模板自家的 `./tasks.sh validate` 兜底。
>
> 占位符 `{{...}}` 与 [governance-snippets/README.md](../governance-snippets/README.md) 「占位符约定」表对齐，下游按自家栈替换（如 `{{validate命令}}` → `pnpm validate` / `make test`）。

## TDD 判定

判据是：**把实现改错时，对应测试会失败**。行为变更类代码默认需要测试，仅以下情形可豁免：

- 仅注释或文档字符串。
- 类型签名内部整理，且不改公共类型契约。
- 无行为重命名。
- 纯常量。
- 纯 re-export。
- 不超过 3 行的纯路径拼接。

禁止为依赖库固有能力写测试（如解析器自身的 parse、内建数组 push 等）。写测试前自问：「实现写错时它会失败吗？」 不会则别写、别留。

## 新功能 / 重构调研

新功能或多文件重构前，先调研同类成熟的开源 / 商用项目，明确行业做法和最佳实践，**避免闭门造车**。

仅以下三类客观条件可降级。**降级是指改用可验证的本地资料、有限公开资料或明确假设完成方案，不等于省略调研**：

- 用户明确限时（如「30 分钟内出方案」）。
- 环境不可用（无网络、外部源访问被拒、网页抓取失败）。
- 已有可复用的本地调研资料（如 `docs/` 下相关文档已覆盖该领域）。

降级时必须记录触发条件和本地依据；**不得自行判断「时间紧」作为跳过理由**。

## 文件大小

明显超大的实现或测试文件需评估是否拆分；**判断依据是单一职责与可读性，而非具体行数**。Schema 集中定义、递归渲染器、生成产物等单一职责文件不必强行拆分。

## 分层验证策略

验证分为开发内循环和收口闸门，目标是保留最终可信证据，同时避免每个小修复重复运行完整套件。

- **RED / GREEN 内循环**：优先运行**受影响的目标测试**。例如修改路由器时先跑对应路由 test；修改 schema 或契约时同时跑对应 contract / schema 测试。
- **review fix 内循环**：先跑覆盖该反馈的目标测试；类型或公共接口受影响时加跑 `{{typecheck 命令}}`；文档链接或 AGENTS 导航受影响时加跑 `{{check命令}}`；AGENTS 结构受影响时加跑 `{{lint命令}}`。
- **任务 / Phase 收口**：在 spec review 和 code quality review 都通过后，运行一次 `{{validate命令}}`，作为该阶段的完整证据。
- **最终交付**：主流程必须**亲自运行**收口验证并读取退出码，**不能只引用 subagent 报告**。
- **高风险改动**：契约、构建配置、跨端边界、共享基础设施、状态布局、协议帧或错误 schema 变更，可在 review fix 后**提前**运行 `{{validate命令}}`，但同一 Phase 不应因每个小修复机械重复全量验证。
- **派发 subagent 时**：prompt 必须写明本次任务的**目标测试**和**收口验证要求**；不要默认要求每个窄修复都运行完整 `{{validate命令}}`，除非该修复本身触发高风险条件。

## 依赖策略

- 功能开发**默认先复用现有依赖**；现有依赖不足时，优先选择成熟、维护活跃、与项目栈兼容良好的社区包。
- 只有在需求很小、引包明显过重、或没有合适成熟包时，才允许自实现通用能力。
- 对解析、校验、日期处理、CLI、重试、日志、schema、HTTP client、文件匹配等通用能力，**默认不要重复造轮子**。
- 新增依赖时，提交说明必须写清**选择原因、至少一个备选方案及未采用原因、运行时/维护成本/复杂度权衡**。
- 包管理器 lockfile（如 `package-lock.json` / `pnpm-lock.yaml` / `yarn.lock` / `Cargo.lock` / `poetry.lock` / `uv.lock`）**必须提交**，禁止 gitignore。
- 依赖升级需**单独提交**，提交说明注明升级原因和影响范围。
- 禁止在功能 PR 中混入无关的依赖升级。
